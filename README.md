# copilot

Repository-level guidance for AI-assisted development.

## Purpose

This repository contains global Copilot instructions used to guide behavior, code quality, security, testing, and documentation practices for AI-driven coding tasks.

## Repository Contents

- `./copilot-instructions.md` — canonical instruction set for agent behavior and engineering standards.

## How to Use

1. Read `copilot-instructions.md` before starting implementation work.
2. Follow the listed standards when proposing, editing, and reviewing code.
3. Keep changes minimal, focused, and aligned with existing project patterns.

## Documentation Standards for AI Usage

- Keep instructions explicit, actionable, and unambiguous.
- Prefer concise rules with clear scope (e.g., coding, testing, security).
- Update documentation whenever behavior expectations change.
- Avoid contradictory guidance across instruction files.

## Contributing

- Use conventional commits (for example: `docs: update AI usage guidance`).
- Do not commit secrets or generated artifacts.
- Preserve this README and `copilot-instructions.md` as the source of truth for repository guidance.
